========================================
  Gemini API 地区限制问题说明
========================================

问题现象：
----------
- 能访问 gemini.google.com 并正常对话
- 但访问 API 页面（makersuite.google.com）会自动跳转到：
  https://ai.google.dev/gemini-api/docs/available-regions

原因：
------
根据 Google 官方文档，Gemini API 和 Google AI Studio 在部分地区不可用。
中国大陆不在支持列表中。

官方文档：
https://ai.google.dev/gemini-api/docs/available-regions

========================================
  解决方案
========================================

方案1：使用支持地区的 API Key（推荐）
----------------------------------------
1. 使用 VPN/代理访问以下地区之一：
   - 美国
   - 日本
   - 新加坡
   - 香港（台湾）
   - 其他支持地区

2. 在支持地区访问：
   https://makersuite.google.com/app/apikey

3. 创建新的 API Key（必须在支持地区访问时创建）

4. 使用新创建的 API Key

注意：即使之后使用代理，如果 API Key 是在受限地区注册的，
Gemini API 仍可能检测到并拒绝访问。

方案2：使用 Vertex AI Gemini API
----------------------------------------
如果无法在支持地区创建 API Key，可以使用 Vertex AI：

1. 注册 Google Cloud Platform (GCP) 账号
2. 启用 Vertex AI API
3. 使用 Vertex AI 的 Gemini API（配置更复杂）

参考：https://cloud.google.com/vertex-ai

方案3：使用本地分析功能（已实现）
----------------------------------------
本系统已包含本地简单分析功能：
- 当 Gemini API 不可用时自动使用
- 基于价格和关键词进行简单评分
- 无需 API Key，完全本地运行

========================================
  常见问题
========================================

Q: 为什么能访问 gemini.google.com 但不能用 API？
A: 网页版和 API 有不同的地区限制策略。
   网页版可能通过代理访问，但 API 会检测 API Key 的注册地区。

Q: 使用代理后为什么还是不行？
A: Gemini API 可能通过以下方式判断地区：
   1. API Key 的注册地区（主要）
   2. 请求的 IP 地址（次要）
   即使代理 IP 正常，如果 API Key 在受限地区注册，仍可能被拒绝。

Q: 如何确认 API Key 是在支持地区注册的？
A: 在创建 API Key 时，确保：
   1. 使用支持地区的 VPN/代理
   2. 访问 https://makersuite.google.com/app/apikey
   3. 如果自动跳转到地区限制页面，说明当前地区不支持

Q: 可以使用现有的 API Key 吗？
A: 如果现有 API Key 是在受限地区创建的，可能无法使用。
   建议在支持地区重新创建新的 API Key。
